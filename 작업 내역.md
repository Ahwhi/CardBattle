## 2026-06-21 - UI 로고 반짝이 쉐이더 제작

### 프롬프트 요약
UI 로고 이미지에 적용할 반짝반짝 쉐이더 하나 만들어줘. 모바일(안드로이드) 고려해서.

### 작업 결과

#### 신규 생성 파일
- **Assets/Shaders/UI_Sparkle.shader**: Unity UI 전용 반짝이 쉐이더

#### 쉐이더 기능
1. **Sweep(빛 스쳐지나감)**: 사선으로 밝은 띠가 주기적으로 로고 위를 지나가는 효과
2. **Sparkle(반짝임)**: 4방향 십자별 형태의 반짝 포인트가 UV 그리드 위에서 랜덤 주기로 명멸

#### 모바일 최적화
- `#pragma target 3.0` (OpenGL ES 3.0)
- 모든 연산에 `half` 정밀도 사용
- 텍스처 샘플: 베이스 텍스처 1회만 샘플링
- 스파클 루프: 2x2(4번) UNITY_UNROLL 언롤 처리
- 지수 연산 최소화 (pow 대신 중복 곱셈)

#### 사용 방법
1. Unity 에디터 Project에서 `Shaders/UI_Sparkle.shader` 확인
2. 새 Material 생성 후 Shader를 `UI/SparkleEffect`로 지정
3. Texture에 로고 이미지(알파 있는 PNG) 할당
4. 로고 Image 컴포넌트의 Material에 해당 Material 할당

#### Inspector 조정 가능 파라미터
| 파라미터 | 설명 | 기본값 |
|---|---|---|
| Sweep Color | 빛 띠 색상 | 밝은 노란빛 흰색 |
| Sweep Width | 빛 띠 폭 | 0.12 |
| Sweep Interval | 스윕 반복 간격(초) | 4.0 |
| Sweep Tilt | 빛 띠 기울기 (0=수평, 1=45도) | 0.45 |
| Sparkle Color | 별 반짝임 색상 | 밝은 황금빛 |
| Sparkle Intensity | 반짝임 밝기 | 1.0 |
| Sparkle Speed | 명멸 속도 | 3.0 |
| Sparkle Density | 반짝임 개수 (그리드 밀도) | 7.0 |
| Sparkle Size | 별 팔 길이 | 0.12 |

궁수 즉사(HP 2이하)는 스킬 사용 시에만 적용. 기본공격에 패시브처럼 적용되면 안됨. 데미지 팝업에서 -/+ 기호 제거, 피해 텍스트를 빨간색으로 변경.

### 작업 결과

#### 수정된 파일
- **TurnManager.cs**: ApplyDamage에 isSkill 파라미터 추가. 스킬 사용이 아닌 기본공격 시 궁수 즉사/레슬러 기절/기동대 출혈/기마병 스킬봉인 등 모든 스킬 고유 효과가 적용되지 않도록 수정 (isSkill=false면 데미지만 적용 후 return).
- **DamagePopup.cs**: 텍스트에서 -/+ 기호 제거 (숫자만 표시). 피해 색상 Color.white -> Color.red 변경.

## 2026-06-22 - 힐러 패시브 밸런스 조정

### 프롬프트 요약
힐러(간호사/의무병) 턴 시작 패시브가 모든 아군 회복은 오버밸런스. 체력이 가장 낮은 아군 1명만 치료하도록 변경. 전원 풀피면 치료하지 않음.

### 작업 결과

#### 수정된 파일
- **TurnManager.cs**: ProcessTurnStartEffects 힐러 패시브 로직 변경. 모든 아군 회복 -> HP가 MaxHP 미만인 아군 중 가장 낮은 1명만 회복. 대상 없으면(전원 풀피) 스킵.

## 2026-06-22 - 총잡이 다중 피격 사운드 수정 / 궁수 즉사 텍스트 추가

### 프롬프트 요약
총잡이 스킬로 여러명 피격 시 첫 번째 대상만 효과음 나오는 버그 수정. 궁수 스킬 즉사 시 KILL 텍스트 팝업 추가.

### 작업 결과

#### 수정된 파일
- **TurnManager.cs**: ExecuteGunnerSkill에서 PlaySfx(Hit)을 foreach 루프 밖 -> 안으로 이동하여 각 대상마다 효과음 재생. ApplyDamage 궁수 즉사 시 SpawnTextPopup("KILL", 노란색) 추가.
- **DamagePopup.cs**: Setup(string text, Color color) 오버로드 추가. 기존 Setup(int, bool)은 내부적으로 PlayPopup 공통 메서드 호출.
- **BattleField.cs**: SpawnTextPopup(RuntimeCard, string, Color) 메서드 추가. 기존 DamagePopup 프리팹을 재활용하여 텍스트 팝업 생성.
