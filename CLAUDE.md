2. 씬
- 타이틀씬: 
1) 상점 패널이 열리는 버튼, 도감 패널이 열리는 버튼
각 패널은 닫기 버튼이 있어서 패널을 닫을 수 있어야함
(도감 패널은 CardDB에는 총 8개의 카드가 있는데 그걸 CardPrefab으로 패널 그리드에 배치하면됨. 슬라이드 하지말고 한 화면에 전부 볼 수 있게)
(카드들 마다 카드 아래에 현재 보유량을 1/10(별레벨승급조건)으로 텍스트로 출력 해야함)
2) 덱 구성 패널이 열리는 버튼 
(CardDB에는 총 8개의 카드가 있는데, 해당 카드가 게임 첫 시작시 전부 1장씩 지급되고, 바로 덱 구성에서 다 볼 수 있어야함)
(덱 구성은 DB의 모든 카드 중 내가 게임에서 사용할 원하는 카드 6개를 드래그/드롭 방식으로 장착/해제 하는 방식)
(6장의 카드가 장착되어 있지 않으면 덱 구성을 완료해야합니다. 하고 게임 시작이 불가능해야함)
(게임 첫 시작시 카드가 자동으로 지급되고 덱구성도 임의로 해놔야함)
3) 게임 시작 버튼(누르면 난이도 선택이 있고 쉬움, 보통, 어려움 난이도 선택 가능, 누르면 바로 해당 게임 난이도로 게임씬으로 이동)
4)게임 종료 버튼(정말 종료하시겠습니까? 패널이 나온 후 확인/취소가 가능하고, 확인 버튼을 누르면 앱종료)

- 게임씬:
1) 게임 개요: 플레이어와 상대방(컴퓨터)이 번갈아 턴을 진행하는 세로형 턴제 카드 배틀 게임
각 플레이어는 여러 장의 카드를 보유하며, 전장에 배치된 카드를 사용해 상대 카드를 공격
상대방의 모든 카드를 제거하면 승리

2) 카드 배치 규칙
게임 시작 시 플레이어와 상대방은 각각 6장의 카드를 보유
3장의 카드는 전장에 배치되어 공개 상태가 되며, 나머지 카드는 뒤집힌 상태의 대기 카드로 존재
전장에 빈 슬롯이 발생하고 아직 공개되지 않은 대기 카드가 남아 있다면, 자동으로 대기 카드 중 하나가 새로운 카드가 되어 
전장에 배치
카드는 CardPrefab으로 생성하면되고, 해당 프리팹에 Card.cs 스크립트가 컴포넌트로 붙어 있고
해당 스크립트의 변수에 맞는 이미지나 글자, 별레벨 이미지등을 프리팹 생성시 수정하면 됨.

3) 턴 진행 규칙
자신의 턴에는 아래 순서로 행동
첫번째: 사용할 아군 카드를 선택
두번째: 공격 또는 스킬을 선택
세번째: 공격 대상 또는 스킬 대상을 선택
네번째: 효과를 적용
다섯번째: 턴 종료
상대방 턴은 자동 진행 또는 간단한 AI 로직으로 구현

4) 체력 규칙
모든 카드는 체력(HP)가 있음
카드 HP가 0 이하가 되면 해당 카드 제거
카드가 제거된 후 대기 카드가 남아 있다면 새로운 카드가 전장에 자동 배치

5) 승리 조건
상대방의 모든 카드가 제거되면 승리
플레이어의 모든 카드가 제거되면 패배

6) 화면
- 세로형이고 내쪽 진영이 아래, 상대쪽 진영은 위
- 일반/힐러/무쌍 카드가 공격시 상대 카드에게 접근하며 공격(DOTWeen 애니메이션 활용)
- 원거리 카드는 살짝만 앞으로 나온 후 VFX로 공격 연출.
- 피격, 스킬시 UniRx 혹은 이벤트형(적합하다고 판단되는 것으로) 반응형 파티클 VFX 적용.
(파티클 프리팹 필요시 작업 완료 후 보고)
- 선택한 카드는 특정 화면 공간에 UI로 해당 카드의 스킬 설명을 출력하는 공간이 있음
- 공격, 스킬 사용 등 버튼이 화면에 배치되어 있음
- 현재 무슨 일을 해야할지 (ex: 카드를 선택하세요. 행동을 선택하세요. 대상을 선택하세요. 등) 안내하는 텍스트도 있음.
- 데미지 적용시 데미지 프리팹이 피격혹은 사용대상위에 나와야함. DOTWeen 애니메이션 활용)
 
3. 카드

일반: 별레벨만큼 상대 카드에게 피해를 준다.
공격한 카드는 상대 별레벨만큼 반격 피해를 받는다.
기본체력: 10

원거리: 별레벨만큼 상대 카드에게 피해를 주지만, 반격 피해를 받지 않는다.
기본체력: 5

무쌍: 별레벨만큼 상대 카드에게 피해를 준다.  인접한 적카드가 있다면, 인접한 적 중 무작위 1장에게
같은 피해를 준다.
기본체력: 5

힐러: (패시브) 힐러가 배치되어 있을 경우 자신의 턴 시작시 아군들의 체력을 별레벨만큼 회복시킨다.
공격은 '일반' 카드와 동일하게 작동한다.
기본체력: 5


전사 / 일반 / 반격 피해를 받지 않고 공격한다.
레슬러 / 일반 / 공격후 1턴간 행동불능 상태로 만든다.


궁수 / 원거리 / 공격 후 적의 체력이 2이하면 즉사시킨다.
총잡이 / 원거리 / 모든 적에게 같은 데미지로 공격한다.


기동대 / 무쌍 / 적 공격 후 2턴간 1의 피해를 받는 출혈 효과를 적용한다.
기마병 / 무쌍 / 공격 적중 후 2턴간 해당 캐릭터가 스킬을 사용하지 못하도록 만든다.


의무병 / 힐러 / 선택한 캐릭터의 체력을 2 회복시키고 모든 상태 이상을 해제한다.
간호사 / 힐러 / 모든 캐릭터의 체력을 1 회복시킨다.


별레벨: 상점같은곳에서 같은 카드 10장 모으면 별레벨 승급

카드들은 CardData.cs의 ScriptableObject로 제작해놨으며
CardDatabase.cs의 ScriptableObject에 할당해둠.

전투시 상태 이상(행동 불능, 출혈, 스킬 봉인)등 효과가 적용된 카드는 카드 아래에 UI로 해당 상태와 남은 상태이상 턴 출력.

4. 상점
패널내에
1) 1장 구매: 100골드 / 무작위 1장
2) 10장 구매: 900골드 / 무작위 10장
버튼이 있고, 해당 버튼 클릭시 CardPrefab이 뒤집어진 상태로 나오고, 클릭시 뒤집혀서 내용물이 공개.
CardPrefab내에 별레벨은 상점에서 보여줄 필요 없으니 starsRoot를 비활성화.

5. 필수 구현 사항
- 턴 진행, 카드 선택(선택시 카드가 살짝 커지고 화살표 이미지 혹은 테두리 이미지가 생기도록), 공격 및 스킬 처리,
카드별 효과 처리, HP 계산, 카드 제거 처리(카드 제거시 DOTWeen 애니메이션 적용),
신규 카드 자동 배치, 승리 및 패배 판정, 간단한 상대 AI, 현재 턴 표시 UI, 카드 HP 표시(UGUI 슬라이더 + 텍스트로 1/5 같은 식으로 최대체력과 함께 표기),
승리 및 패배 결과 패널

7. 코딩 규칙
- 범용적으로 유지 및 보수가 편한 코드를 작성하시오.
- 주석에 특수문자 및 유니코드를 사용하지 마시오.
- 일부 기능을 위해 범용성 없는 헬퍼 함수나 헬퍼 스크립트를 만들지 마시오.
- 객체 지향의 의미를 가지고 역할 분담이 명확한 디커플링 코딩을 하시오.
- 파일 수정 전 인코딩을 확인할 것.
* Edit 툴은 UTF-8 전용이다. EUC-KR 파일을 Edit 툴로 수정하면 한글이 전부 파괴된다.
   EUC-KR 파일 수정 전 반드시 PowerShell로 UTF-8 변환 먼저 할 것:
   $euckr = [System.Text.Encoding]::GetEncoding(949)
   $utf8 = New-Object System.Text.UTF8Encoding($false)
   [System.IO.File]::WriteAllText($out, [System.IO.File]::ReadAllText($in, $euckr), $utf8)
신규 파일은 처음부터 UTF-8로 저장할 것.
- 내가 쓰지 않았지만 위 기능들을 구현하는데 필요하다고 판단되면 알아서 구현 후 작업 종료시 나에게 보고 할 것
- 작업 종료 후 유니티 에디터 상에서 내가 구현해야할 UI, 인스펙터 할당 작업, 스크립터블 오브젝트 작업등을 리스트업해서 보고 할 것
